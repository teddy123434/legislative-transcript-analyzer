# GUI 版本使用說明

## 啟動 GUI

```bash
python3 gui.py
```

## 委員名單檔案格式

建立一個 `speakers.txt` 檔案，每行一個委員名字：

```
主席
陳委員培瑜
王委員義川
翁委員曉玲
李委員昆澤
```

## 使用流程

### 1. 匯入委員名單
- 方式 A：點擊 **📁 匯入名單 (txt)**，選擇你準備好的 txt 檔（每行一個名字）。
- 方式 B：點擊 **📝 貼上名單**，在彈出視窗貼上名單（每行一個名字），按下「套用」。

### 2. 載入逐字稿
三種方式選一：
- **📄 讀取 Word (.docx)** - 直接讀取 Word 檔案
- **📋 讀取文字檔 (.txt)** - 讀取文字檔案
- **或直接貼上** - 在左側文本框貼上逐字稿內容

### 3. 解析
- 點擊 **✓ 解析文本** 按鈕
- 程式會自動統計符合名單中的委員發言

### 4. 查看結果
- 右側「統計結果」會顯示所有委員的發言次數和字數
- 點擊 **🔍 查詢特定委員** 可看該委員的詳細發言內容

### 5. 匯出
- **📊 輸出 Excel**：匯出目前載入的單一檔案，包含摘要表＋每位委員詳細分頁。
- **📑 多檔分表匯出**：一次選擇多個 .docx/.txt，匯出到同一個 .xlsx；每個檔案一個工作表（摘要表），不產生委員詳細分頁，避免過多工作表。

### 5. 輸出
- 點擊 **📊 輸出 Excel** 可匯出統計報表

## 功能按鈕說明

| 按鈕 | 功能 |
|------|------|
| 📁 匯入名單 | 讀取 txt 檔案的委員名單 |
| 🗑️ 清除限制 | 移除委員限制，統計所有發言人 |
| 📄 讀取 Word | 載入 .docx 檔案 |
| 📋 讀取文字檔 | 載入 .txt 檔案 |
| ✓ 解析文本 | 解析輸入的逐字稿內容 |
| 🔍 查詢特定委員 | 查看某位委員的詳細發言 |
| 📊 輸出 Excel | 匯出統計結果到 Excel |
| 🔄 刷新 | 刷新統計結果顯示 |

## 委員名單格式

委員名單必須與逐字稿中的名字完全相同，例如：

✓ **正確格式**：
```
主席
陳委員培瑜
王委員義川
```

❌ **錯誤格式**：
```
陳培瑜        # ← 缺少「委員」
王義川        # ← 缺少「委員」
```

## 逐字稿格式

確保逐字稿採用以下格式（支援「：」和「:」）：

```
主席：條文已宣讀完畢，現在進行討論。
陳委員培瑜：沒有意見。
王委員義川：同意。
```

### 多行續接支援 ✨

**好消息**：程式已支援多行續接！當發言人的話被分成多行（沒有「名字：」前綴），會自動合併。

範例：
```
主席：提案條文都已經宣讀完畢，請問各位委員有無異議？
再來我們進行逐條討論協商。
報告委員會，修正動議已撤案。
黃委員捷：謝謝，我覺得這個版本可以。
```

第2、3行會自動計入主席的發言中，無需手動合併。

## 提示

1. **建議先匯入委員名單** - 避免誤抓不相關的發言人（如「機關代表」等）
2. **批量處理多個檔案** - 可以連續匯入多個逐字稿，會累計統計
3. **查詢委員詳情** - 可以看到該委員的每一句發言
4. **Excel 檔會自動建表** - 每位委員一個工作表

## 介面配色與 macOS 注意事項

- 本工具在 macOS 上改用 ttk 的 `clam` 主題，避免原生 Aqua 主題導致按鈕顏色被覆蓋成白色。
- 主要按鈕顏色：藍色（主要動作）、綠色（匯入/確認）、紅色（清除/危險）、灰色（刷新/次要）。
- 若您仍看到按鈕是白色，請更新系統的 Python/Tk 或重新啟動程式再試。

### 深色模式
- 右上角按鈕 **🌗 主題** 可切換淺色/深色。
