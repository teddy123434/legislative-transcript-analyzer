# CCW 逐字稿分析工具使用說明

## 功能
分析公督盟委員會審查會議逐字稿，統計各委員的：
- 發言次數
- 發言字數
- 平均每次發言字數
- 所有發言內容詳情

## 支援的檔案格式
- Word 檔案：`.docx` ✓
- 文字檔案：`.txt` ✓
- 直接貼上文本 ✓

## 使用方法

### 方式 1：互動式選單（推薦）
```bash
python3 interactive.py
```

選單中可以：
1. **設定委員名單** - 指定要統計的委員（避免誤抓非委員發言）
2. **清除委員限制** - 統計所有發言人
3. 讀取 Word 檔案 (.docx)
4. 讀取文字檔案 (.txt)
5. 貼上逐字稿文本
6. 查看所有委員統計
7. 查看特定委員統計
8. 輸出到 Excel

### 方式 2：命令行直接處理
```bash
# 讀取 Word 檔案
python3 interactive.py meeting.docx output.xlsx

# 讀取文字檔案
python3 interactive.py meeting.txt output.xlsx
```

### 方式 3：Python 代碼
```python
from ccw_analyzer import CCWTranscriptAnalyzer

# 建立分析器並指定委員名單
speakers = ["主席", "陳委員培瑜", "王委員義川"]
analyzer = CCWTranscriptAnalyzer(speakers)

# 讀取 Word 檔案
analyzer.load_from_word('meeting.docx')

# 或讀取文字檔案
with open('meeting.txt', 'r', encoding='utf-8') as f:
    text = f.read()
analyzer.parse_transcript(text)

# 查看統計
analyzer.print_summary()

# 輸出到 Excel
analyzer.export_to_excel("統計結果.xlsx")

# 設定或清除委員名單
analyzer.set_speaker_list(["主席", "新的委員"])
analyzer.clear_speaker_list()  # 清除限制，抓取所有
```

## 委員名單功能
這是為了避免自動抓取時誤抓非委員的發言（如"機關代表"等）。

**範例**：
```
不指定名單時，會抓取所有：
- 主席
- 陳委員培瑜
- 王委員義川
- 陳次長彥伯  ← 可能誤抓

指定名單後：
- 主席
- 陳委員培瑜
- 王委員義川
（陳次長彥伯 被過濾掉）
```

## 文本格式
逐字稿應採用以下格式（支援兩種冒號）：
```
委員名字：發言內容
主席：條文已宣讀完畢
陳委員培瑜：沒有意見。
王委員義川：同意。
```

### 多行續接支援 ✨
**新功能**：程式現已支援多行續接！當同一個發言人的話被分成多行時（沒有「名字：」前綴的行），會自動接續到該發言人的最後一句。

範例：
```
主席：提案條文都已經宣讀完畢，請問各位委員有無異議？
再來我們進行逐條討論協商。
報告委員會，修正動議已撤案。
黃委員捷：謝謝，我覺得這個版本可以。
後續的討論我們可以再進行。
```

在上面的例子中：
- 第2、3行（沒有「主席：」）會自動計入主席的發言
- 第5行（沒有「黃委員捷：」）會自動計入黃委員捷的發言

## 輸出結果
### 總結工作表 (發言統計)
- 委員名字
- 發言次數
- 總字數
- 平均字數

### 多檔案分表匯出
- 在 GUI 按「📑 多檔分表匯出」可一次選取多個逐字稿檔案（.docx / .txt）。
- 每個檔案會成為一個工作表（名稱取自檔名，超過 31 字會截斷，必要時加序號）。
- 分表匯出僅產出摘要表，避免生成過多詳細分頁。單檔匯出仍會附每位委員詳細內容。

### 詳細工作表 (每位委員一頁)
- 所有發言內容列表
- 標記序號

## 注意事項
- 支援「：」和「:」兩種冒號
- 空白行會被自動忽略
- 統計不計算冒號前的名字部分，只計算發言內容
- Word 檔案會自動提取所有段落文本
- 確保檔案使用 UTF-8 編碼
- **建議先指定委員名單，避免誤抓其他發言人**
